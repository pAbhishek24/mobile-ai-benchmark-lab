Methodology Notes

v1 (legacy) — Runs collected before methodology versioning was introduced. No explicit benchmark_mode tag. Conditions: background apps active, no post-reboot, screen on, One UI thermal management. These runs represent real-world consumer conditions but lack controlled baselines for comparison.

v2 — Runs using the upgraded framework. Each run explicitly declares benchmark_mode, post_reboot, airplane_mode, and other environment metadata. Both real_world and controlled runs are captured per model for direct comparison.

real_world — Background apps active, no forced reboot, One UI thermal throttling engaged. Represents typical consumer device conditions. Latency will be higher and more variable due to OS interference.

controlled — Airplane mode on, fresh reboot, no background apps (to the extent possible), stable thermal baseline. Represents clean-room conditions for benchmarking model performance without OS noise.
